= Using Kubeflow Trainer v2 to run distributed training workloads


[role='_abstract']

You can use Kubeflow Trainer v2 to run distributed PyTorch training workloads on {productname-long}. This section describes how to create a TrainJob resource by using the OpenShift Console or the OpenShift CLI (`oc`).

A TrainJob resource references a ClusterTrainingRuntime (or custom TrainingRuntime) that provides the distributed training infrastructure, so you only need to specify your training code, the number of nodes, and resource requirements.
