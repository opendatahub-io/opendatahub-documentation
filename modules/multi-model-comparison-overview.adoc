= How multi-model comparison accelerates model selection

[role="_abstract"]
You can test multiple AI models side-by-side with identical prompts using multi-model comparison.
Comparing model responses in parallel helps you identify the best-performing model for your use case based on quality, latency, and behavior.

When you enable multi-model comparison in the GenAI Playground, the interface displays responses from multiple models simultaneously.
Each model receives the same prompt with identical parameters, enabling direct comparison of outputs.

== Benefits of multi-model comparison

Accelerates model selection::
You can evaluate multiple models in a single session instead of configuring separate playground instances for each model.

Reveals performance differences::
Side-by-side comparison highlights variations in response quality, tone, verbosity, and accuracy across models.

Enables A/B testing of prompts::
You can test how different models respond to the same prompt, helping you select models that are robust to prompt variations.

Compares latency and behavior::
You can observe which models generate responses faster and which models handle specific tasks more effectively.

== Common use cases

Multi-model comparison is useful when you need to:

* Evaluate open-source models against commercial models
* Compare different versions of the same model (for example, Llama 3 vs Llama 3.1)
* Test how models respond to complex or ambiguous prompts
* Assess cost-performance tradeoffs for production deployment
* Select a model for a specific domain (for example, code generation vs summarization)

== Limitations in Technology Preview

In the Technology Preview release, multi-model comparison has the following limitations:

* A maximum of three models can be compared simultaneously
* All models must be deployed as AI asset endpoints before comparison
* Shared parameters (temperature, top-k, top-p) apply to all models equally
* Individual model parameters cannot be configured differently during comparison
* Conversation history is shared across all models in the comparison
