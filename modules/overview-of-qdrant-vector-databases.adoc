= Overview of Qdrant vector databases

[role="_abstract"]
Qdrant is an open source vector database optimized for high-performance similarity search and advanced filtering. In {productname-short}, Qdrant is supported as a remote vector store provider for Llama Stack and can be used in retrieval-augmented generation (RAG) workloads that require efficient vector indexing and durable storage.

When used with Llama Stack in {productname-short}, Qdrant provides:

* High-performance similarity search using Hierarchical Navigable Small World (HNSW) indexing
* Filtering based on stored metadata during vector search
* Persistent storage of vector data
* Integration through the OpenAI-compatible Vector Stores API

In a RAG workflow:

* Embeddings are generated by the configured embedding provider.
* Qdrant stores embedding vectors and performs similarity search.
* Llama Stack manages ingestion, retrieval, and model inference through a unified API.

In {productname-short}, Qdrant must be deployed as a remote service, either within your {openshift-platform} project or as an externally managed deployment.

[NOTE]
====
Inline Qdrant is not supported. To use Qdrant with Llama Stack in {productname-short}, deploy Qdrant as a remote service.
====

A typical remote deployment includes:

* A Qdrant service exposing HTTP (port 6333) and gRPC (port 6334) endpoints
* Persistent storage for vector data
* Optional API key authentication

For deployment and configuration instructions, see xref:using-qdrant-in-llama-stack_{context}[Using Qdrant in Llama Stack].
