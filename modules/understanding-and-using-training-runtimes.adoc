= Understanding and using training runtimes


[role='_abstract']


Kubeflow Trainer v2 uses training runtimes to define the distributed training infrastructure for your training jobs.
Runtimes encapsulate best practices for distributed training, including PyTorch distributed
configuration, torchrun setup, environment variables, and container image defaults.


This section describes how to view the pre-built ClusterTrainingRuntime resources provided by Red Hat OpenShift AI, and how
to create custom namespace-scoped TrainingRuntime resources when you need project-specific configuration.
