= Working with hardware profiles

[role='_abstract']
ifndef::upstream[]
[IMPORTANT]
====
Hardware profiles are currently available in {productname-long} as a Technology Preview feature only. Technology Preview features are not supported with {org-name} production service level agreements (SLAs) and might not be functionally complete. {org-name} does not recommend using them in production. These features provide early access to upcoming product features, enabling customers to test functionality and provide feedback during the development process.

For more information about the support scope of {org-name} Technology Preview features, see link:https://access.redhat.com/support/offerings/techpreview[Technology Preview Features Support Scope]
====
endif::[]

In {productname-long}, you can manage and allocate specific hardware resources, such as hardware accelerators, specialized memory, or CPU-only nodes for data science, machine learning, and generative AI workloads using *hardware profiles*. 

A hardware profile is a custom resource (CR) that you create in {productname-short} to specify a particular hardware configuration. You can create and manage these profiles by going to *Settings* -> *Hardware profiles* on the {productname-short} dashboard.

[IMPORTANT]
====
By default, this feature is hidden from appearing in the dashboard navigation menu. To show the *Settings -> Hardware profiles* option in the dashboard navigation menu, and other user interface components associated with hardware profiles, set the `disableHardwareProfiles` value to `false` in the `OdhDashboardConfig` custom resource (CR) in {openshift-platform}.
ifdef::upstream[]
For more information about setting dashboard configuration options, see link:{odhdocshome}/managing-resources/#customizing-the-dashboard[Customizing the dashboard].
endif::[]
ifndef::upstream[]
For more information about setting dashboard configuration options, see link:{rhoaidocshome}{default-format-url}/managing_resources/customizing-the-dashboard[Customizing the dashboard].
endif::[]
====

After you create and enable a hardware profile, you can select it when you deploy resources such as workbenches, model-serving workloads, and pipelines. This capability enables precise targeting of your workloads to specific nodes that have the required hardware, which ultimately improves resource management efficiency.

Hardware profiles offer comprehensive control over resource allocation by allowing you to define profiles with hardware identifiers, explicit resource limits (for CPU, memory, and accelerators), tolerations, and node selectors.

This detailed control is extremely useful in heterogeneous environments (clusters with diverse hardware, like multiple GPU types, CPU-only nodes, or memory-intensive setups). By enabling this targeted scheduling, hardware profiles significantly enhance resource utilization, reduce overhead, and ultimately optimize costs.

To get started, contact your cluster administrator to identify hardware resources available in your cluster.
